## 📝 文档规范

### 文档命名规范

- **API 接口文档**：`模块名称 API 接口文档.md`
- **功能实现文档**：`功能名称功能实现说明.md` 或 `功能名称实现总结.md`
- **问题修复记录**：`问题描述修复说明.md` 或 `问题描述修复完成报告.md`
- **使用指南**：`功能名称使用指南.md` 或 `功能名称快速参考.md`

### 文档内容规范

每个文档应包含：

1. **标题**：清晰的文档标题
2. **概述**：功能或问题的简要说明
3. **详细内容**：具体的实现、使用方法或解决方案
4. **示例代码**：必要时提供代码示例（使用代码块，不创建代码文件）
5. **注意事项**：重要提示和注意事项
6. **版本历史**：重要更新记录

### 代码示例规范

✅ **推荐做法**：在 Markdown 文档中使用代码块

```typescript
// 示例代码写在Markdown文档中
const example = {
  // ...
};
```

❌ **不推荐做法**：在 docs 目录下创建独立的代码文件（.vue, .ts, .js 等）

- 代码文件应该放在 `src/` 目录下
- 文档只保留 Markdown 格式文件
